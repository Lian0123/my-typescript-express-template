# 文件生成與維護索引

## 文件角色

| 文件 | 寫什麼 | 何時更新 |
|---|---|---|
| `入口.md` | 專案摘要與任務路由 | 新增功能域、主要技術或入口改變 |
| `重點記憶/專案事實.md` | 跨任務仍穩定的已驗證事實 | 架構、資料流、能力邊界改變 |
| `重點記憶/風險與技術債.md` | 會影響決策的已知風險 | 發現、修復或重新評級風險 |
| `快速定位/*索引.md` | 責任、入口、搜尋詞、測試 | 路徑、contract、資料流改變 |
| 任務／規範／驗證索引 | 可重複流程與完成定義 | 工具或品質門檻改變 |
| `異動記錄/CHANGELOG.md` | 對使用者或維護者有意義的變更 | 每次可交付能力／操作變更 |
| `外部參考/` | 外部來源的摘要與取用規則 | 來源新增、失效或版本改變 |

## 每份文件格式

第一行必須是：

```text
<!-- AI-DOC: owner=<責任>; verified=YYYY-MM-DD; sources=<逗號分隔來源> -->
```

正文使用繁體中文；程式符號保留原名。只陳述可驗證現況，計畫標記「提案／待驗證」。以相對連結與穩定符號取代複製原始碼與行號。

## 更新演算法

1. 用 git diff 或任務範圍列出變更檔。
2. 由功能總索引找到所有受影響 contract。
3. 用原始碼／測試／執行結果驗證新事實。
4. 就地更新既有權威文件；不要新增同義文件。
5. 更新 `verified` 日期，只限本次真的重新核對過的文件。
6. 若屬可交付異動，追加 CHANGELOG；純文字修正不需日誌。
7. 執行 `npm run docs:verify`。

大量既有資料轉入本架構時使用 `skills/convert-to-ai-docs`；日常程式維護使用 `skills/maintain-with-ai-docs`。skill 若要被 Codex 全域自動發現，可由維護者安裝／連結至 `$CODEX_HOME/skills`，但 repository 內版本是權威來源。
