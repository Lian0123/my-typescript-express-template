# 風險與技術債

此處只列已從 repository 觀察到、會影響維護判斷的事項。未經任務授權不要順手大改；修正時補測試並更新本頁。

| 等級 | 事項 | 影響／驗證方向 |
|---|---|---|
| 高 | TypeORM 依賴版本為 `0.3.x`，程式仍使用 `createConnection`、`Connection`、`EntityRepository`、`getCustomRepository` 等舊 API | `npm run build` 是最低門檻；升級或修復需整體檢查 data source 與 repository 注入 |
| 高 | `synchronize: true` 與 migrations 同時存在 | 正式環境可能出現 schema 非預期異動；部署前需建立環境化策略 |
| 高 | RabbitMQ consumer 在非同步 handler 完成前就 `ack`，且未處理 handler rejection | 失敗訊息可能遺失；修改事件可靠性時需測 ack/nack 與重試 |
| 中 | `docker-compose*.yaml` 的 Redis 密碼字串疑似缺少結尾引號，部分 Mongo 變數拼字沿用 `UERNAME`、`DATABAE` | 啟動基礎設施前先執行 compose config 並對照 `.env.example` |
| 中 | 聊天 entity／interface 含多個拼字與型別疑點，repository 空白 | 視為未完成雛形，不可宣稱具備持久化聊天功能 |
| 中 | 時序資料目前以標準 PostgreSQL 表為基底，TimescaleDB hypertable 只在 extension 已存在時才會建立 | 如果要把 `time_series_samples` 升級成真正的 hypertable，需先確認資料庫映像檔或伺服器已安裝 TimescaleDB extension |
| 中 | lint 指令包含 `--fix` | 執行會修改檔案；驗證後必須檢查 diff，避免混入非任務格式化 |
| 中 | 測試可能依賴外部連線或舊版 TypeORM mock 行為 | 區分單元測試與 E2E；記錄未執行原因，不以單一通過結果宣稱全系統正常 |
| 低 | CI actions 版本偏舊，README 原先無使用說明 | 升級 CI 時按官方 migration notes 驗證權限與輸出 |

安全底線：不得提交 `envs/.env`、密碼、token、連線字串或真實個資；對外輸入必須經 DTO 驗證；錯誤回應不可洩漏 stack 或基礎設施細節。
