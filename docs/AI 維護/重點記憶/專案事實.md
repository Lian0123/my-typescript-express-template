# 專案事實

## 不讀原始碼也必須知道

| 面向 | 已驗證事實 | 主要來源 |
|---|---|---|
| 執行環境 | Node.js 專案；TypeScript 目標 ES2017、CommonJS | `package.json`、`tsconfig.json` |
| HTTP | Express + `inversify-express-utils` decorator controller | `src/main.ts`、`src/accounts/*.controller.ts` |
| 依賴注入 | `src/container.ts` 建立連線並綁定 controller、service、repository、DTO、AO | `src/container.ts` |
| 關聯式資料 | PostgreSQL + TypeORM；實體為 account、role、time_series_samples | `src/typeorm-config.ts`、`src/accounts/entities/`、`src/timeseries/entities/` |
| 非同步事件 | account／role create、update 透過 RabbitMQ queue | `src/accounts/*.event.controller.ts` |
| 即時通訊 | Socket.IO namespace `/v1/chatting`，事件 `message`、`read` | `src/chatting/chatting.socket.controller.ts` |
| 文件/API | Swagger UI 靜態頁於 `/api-docs` | `src/main.ts`、`src/swagger/index.html` |
| 測試 | Jest + ts-jest；測試位於 `src/test/` | `jest.config.js`、`package.json` |

## 啟動與請求資料流

`src/main.ts` → 載入 `src/env.ts` → 建立 Inversify container → `src/container.ts` 連線外部服務並綁定元件 → 建立 Express 與 Socket.IO server。

HTTP：Controller 驗證 DTO → Service 執行業務規則／發事件 → Repository 存取 TypeORM → AO 轉換回應。

命名語意：DTO 是跨層輸入資料；AO 是 API 輸出；BO 是 service 層形狀；PO 是 repository 層形狀；Entity 是資料庫映射。

## 目前功能邊界

- 帳號：單筆／分頁查詢、建立、更新、軟刪除、更新角色。
- 角色：單筆／分頁查詢、建立、更新、軟刪除。
- 時序資料：可寫入單筆／批次 sample，依時間區間查詢與 bucket 彙總；目前以 PostgreSQL 表與索引實作，migration 會在具備 TimescaleDB extension 時嘗試建立 hypertable。
- 時序資料維護：提供 downsample 到 rollup table 與 retention 刪除 raw data 的管理 API；compose override 可切換至 TimescaleDB 容器。
- 聊天：只有連線、斷線、訊息 echo 與 read acknowledgement 雛形。
- Redis client 已建立但未在主要流程注入；MongoDB 連線已綁定，但聊天 repository 尚未實作。
