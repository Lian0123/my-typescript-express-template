# 日誌索引

- [CHANGELOG](CHANGELOG.md)：只記錄 living documentation 建立後的重要能力、操作、相容性與文件架構異動。
- Git history：程式碼級完整歷史與作者資訊的權威來源。
- Issue／PR：任務脈絡、討論與驗收證據的權威來源（若存在）。

搜尋順序：先查 CHANGELOG 的關鍵字，再用 `git log --all -- <path>` 與 `git blame <path>` 定位原始變更。不要把完整 git 歷史複製進文件。

CHANGELOG 採日期分段，每筆標記 `新增`、`變更`、`修正`、`移除`、`安全` 之一；尚未交付內容放 `未發佈`。
