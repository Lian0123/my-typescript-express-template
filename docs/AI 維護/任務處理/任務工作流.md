# 任務工作流

## 開始

建立簡短工作單，可放在 issue、PR 或暫存筆記，不必為每個任務新增永久文件：

```text
目標：一句話描述可觀察結果
非目標：本次刻意不處理的範圍
證據：錯誤、需求、測試或來源
影響：功能／資料／API／事件／配置
驗證：要執行的指令與人工檢查
文件：需更新的索引／事實／風險／CHANGELOG
```

## 執行

1. 由入口選路，搜尋實際符號與既有測試。
2. 修 bug 時先證明失敗；新功能先定義 contract 與不變條件。
3. 實作最小完整垂直切片，不留下無入口的 dead code。
4. 新增或更新能捕捉需求的測試。
5. 檢查 diff，移除除錯輸出與無關變更。
6. 執行驗證矩陣並記錄限制。
7. 同步 living docs；只有使用者可感知或維護方式改變才寫 CHANGELOG。

## 完成定義

- 需求有可觀察證據。
- build、相關測試、lint／文件檢查均有結果；未執行項有原因。
- 安全、資料相容性與錯誤路徑已考慮。
- 新 AI 能由入口在兩次連結內找到變更位置。
- 文件不含密鑰、虛構能力、過時行號或重複大段程式碼。
