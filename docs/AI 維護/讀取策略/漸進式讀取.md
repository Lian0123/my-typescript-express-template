# 漸進式讀取策略

## 預設讀取預算

1. 入口層：只讀 `入口.md`。
2. 索引層：只讀一份任務索引與一份功能索引。
3. 原始碼層：先用 `rg` 找符號，再讀命中的局部檔案。
4. 擴展層：只有跨模組變更才讀第二份功能索引或風險文件。

停止條件：已能回答「入口在哪裡、資料如何流動、應改哪些檔案、如何驗證」時，停止蒐集並開始工作。

## 建議查詢

```bash
rg -n "關鍵字|類別名|路由" docs/AI\ 維護 src
rg -n "@(controller|httpGet|httpPost|httpPut|httpDelete)" src
rg -n "@inject|bind<|toConstantValue" src/container.ts src
rg -n "process\.env|POSTGRES|RABBIT|REDIS|MONGO" src envs/.env.example docker-compose*.yaml
rg -n "describe\(|it\(|test\(" src/test
```

## 可信度順序

可執行結果 > 目前程式碼與設定 > 測試 > AI 維護文件 > README／歷史記錄 > 註解與推測。

- 文件與程式不符：標記為文件漂移，驗證程式後更新文件。
- 測試與需求不符：先確認需求，再同時修正實作、測試與文件。
- 找不到證據：明寫「未知／待驗證」，不可補成事實。

## 降低 token 的寫作規則

- 一個事實只保留一個權威位置，其他文件用相對連結。
- 索引寫「去哪裡找、何時讀」，不複製完整實作。
- 以檔案與穩定符號定位，不依賴易漂移的行號。
- 只保存穩定決策、限制與驗證方式；不保存對話過程。
- 長清單以表格或可搜尋關鍵字表示，避免敘事重複。
