# 安裝與配置索引

## 本機基本流程

```bash
npm ci
cp envs/.env.example envs/.env
npm run docker-compose:local
npm run migration:run
npm run start:dev
```

這是預期流程，不代表目前 compose 與 TypeORM 相容性已完全驗證；遇錯先讀 [風險與技術債](重點記憶/風險與技術債.md)。`npm ci` 會觸發 `preinstall`／`prepare`，可能調整 script 權限與安裝 Husky。

## 主要配置來源

| 類別 | 權威位置 | 使用者 |
|---|---|---|
| 服務 | `envs/.env.example` | `src/main.ts` |
| PostgreSQL | 同上 | `src/typeorm-config.ts`、compose |
| RabbitMQ | 同上 | `src/container.ts`、compose |
| MongoDB | 同上 | `src/container.ts`、compose |
| Redis | 同上 | `src/redis/index.ts`、compose |
| 測試／編譯／lint | `jest.config.js`、`tsconfig.json`、`.eslintrc.json` | npm scripts |

新增或更名環境變數時，同步 `envs/.env.example`、所有 consumer、compose、此索引與測試。範例只放無敏感預設值；實際 `.env` 不提交。

## 常用指令

- `npm run build`：TypeScript 編譯並調整 dist TypeORM 路徑。
- `npm run start:dev`／`start:prod`：開發／編譯後啟動。
- `npm run migration:create -- <名稱>`、`migration:run`、`migration:revert`：schema migration。
- `npm run env:change -- <環境>`：切換 env 檔；會覆寫 `envs/.env`，執行前確認備份。
- `npm run db:diagram`、`openapi:generate`：需外部服務／Docker，不能視為純本機檢查。
