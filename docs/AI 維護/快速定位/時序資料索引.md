# 時序資料索引

## 路徑與責任

| 層 | 路徑 | 責任 |
|---|---|---|
| HTTP | `src/timeseries/time-series.controller.ts` | 時序樣本寫入、查詢、彙總 |
| 業務 | `src/timeseries/time-series.service.ts` | 驗證範圍、轉派 repository、整理回應 |
| 資料 | `src/timeseries/repositories/time-series.repository.ts` | 時序 sample 的新增、區間查詢、bucket 彙總 |
| DB | `src/timeseries/entities/time-series-sample.entity.ts`、`src/migrations/1783641600000-time-series-samples.ts` | `time_series_samples` 表與索引 |
| 輸入／輸出 | `dto/`、`ao/`、`bo/`、`po/` | request 驗證與 response 形狀 |
| 測試 | `src/test/timeseries/time-series.service.spec.ts` | service 與 contract 行為 |

## HTTP contract 快覽

- `POST /v1/time-series/samples`: 寫入單筆 sample。
- `POST /v1/time-series/samples/batch`: 寫入多筆 sample。
- `GET /v1/time-series/samples`: 依 `seriesKey`、時間區間、`metricName`、`limit`、`offset` 查詢。
- `GET /v1/time-series/summary`: 依 `bucket` 對同一查詢條件做彙總。

## 時序處理不變條件

- 任何查詢都必須帶 `seriesKey`、`startAt`、`endAt`。
- 時間區間採半開區間 `startAt <= recordedAt < endAt`，避免跨桶重疊。
- `bucket` 僅接受 `minute`、`hour`、`day`、`week`、`month`。
- repository 以 PostgreSQL 為基礎，migration 會在偵測到 TimescaleDB extension 時嘗試建立 hypertable，但不是硬性依賴。

## 修改檢查

新增欄位或索引時，先改 entity + migration，再改 repository query，最後補 controller DTO、AO 與測試。
