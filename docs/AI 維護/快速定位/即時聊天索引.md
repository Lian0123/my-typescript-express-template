# 即時聊天索引

## 現況

- `src/chatting/chatting.socket.controller.ts`：namespace `/v1/chatting`；處理 connect、disconnect、`message`、`read`。
- `message` 目前只記錄 payload 並回傳 `Hello!`；`read` 回傳 `{ status: 200 }`。
- `src/chatting/chatting.repository.ts` 與 HTTP controller 尚未實作。
- `src/chatting/chatting.entity.ts` 有 Mongo 風格 ObjectID 模型，但主要 TypeORM config 是 PostgreSQL glob；實際持久化整合需重新驗證。
- `src/redis/index.ts` 建立 Redis client，但未由 container 注入聊天流程。
- `src/test/chatting/chatting.socket.controller.spec.ts` 是現有測試入口。

3. 身分驗證、房間授權、重連、重複訊息與順序如何處理？
