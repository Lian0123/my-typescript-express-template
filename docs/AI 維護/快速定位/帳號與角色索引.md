# 帳號與角色索引

## 路徑與責任

| 層 | 帳號 | 角色 |
|---|---|---|
| HTTP | `account.controller.ts` | `role.controller.ts` |
| 業務 | `account.service.ts` | `role.service.ts` |
| 資料 | `repositories/account.repository.ts` | `repositories/role.repository.ts` |
| DB | `entities/account.entity.ts` | `entities/role.entity.ts` |
| 輸入／輸出 | `dto/`、`ao/`、`bo/`、`po/`、`interfaces/` | 同左 |
| 事件 | `account.event.controller.ts` | `role.event.controller.ts` |
| 測試 | `src/test/accounts/account*.spec.ts` | `src/test/accounts/role*.spec.ts` |

以上路徑皆相對 `src/accounts/`，測試除外。

## HTTP contract 快覽

- `/v1/accounts`: GET list、POST create；`/:id`: GET、PUT、DELETE；`/:id/role`: PUT。
- `/v1/roles`: GET list、POST create；`/:id`: GET、PUT、DELETE。
- Controller 應用 `validateClass` 將輸入轉為 DTO；回傳資料經 AO 轉換。
- create／update 會發 RabbitMQ 事件；delete 是 `isDeleted` 軟刪除。

## 帳號角色更新的不變條件

- 目標帳號必須存在。
- 所有要求角色都必須存在。
- `isUnique` 角色不能與其他角色並存。
- 角色 `totalCount` 必須依新增／移除差集更新。
- 此流程標示 `@Transactional()`；修改時需驗證 repository 是否真的加入相同 transaction context。

## 修改檢查

DTO／欄位 → controller + Swagger → service → repository/entity/migration → event payload → AO/BO/PO → container binding（若 constructor 改變）→ 對應 unit/E2E tests。
