# 測試與驗證索引

## 驗證矩陣

| 變更 | 最低驗證 | 加強驗證 |
|---|---|---|
| 純文件 | `npm run docs:verify` | 人工點擊入口與相對連結 |
| TypeScript 一般變更 | `npm run build`、相關 spec、`npm run docs:verify` | `npm run lint`、完整 unit tests |
| Controller／API | 上述 + controller spec | `npm run test:e2e`、Swagger 人工檢查 |
| Service／repository／entity | 上述 + service/repository spec | migration run/revert、真實 PostgreSQL 整合 |
| RabbitMQ／Socket／Redis／Mongo | build + 對應 spec | 啟動依賴服務、失敗／重連／重複事件測試 |
| 配置／依賴／Docker | build + docs verify | `docker compose config`、乾淨 `npm ci`、完整測試 |

## 指令語意

```bash
npm run build
npm run test:once
npm run test
npm run test:e2e
npm run lint
npm run docs:verify
```

- `test` 排除 E2E 並收 coverage；`test:once` 會包含 Jest 可找到的全部測試；需要精準範圍時使用 `npx jest <path> --runInBand`。
- `lint` 帶 `--fix`，會修改檔案；執行後檢查 diff。
- E2E 或啟動測試可能需要 PostgreSQL、RabbitMQ、MongoDB 等外部依賴。
- 不得只報「測試通過」；交付時列出實際指令、結果、未執行項與原因。

## 測試品質

測試可觀察行為與不變條件，不鎖死內部實作。每個 bug 至少有一個先失敗後通過的回歸案例；mock 只替代真正邊界，不能讓錯誤的 production DI 或 query 永遠無法被發現。
